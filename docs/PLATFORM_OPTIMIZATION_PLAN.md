### 1.2 文件上传 API（关键前置能力）

Suno 提供 [File Upload API](https://docs.sunoapi.org/file-upload-api/quickstart)：

| 方式 | 端点 | 适用场景 | 限制 |
|------|------|----------|------|
| Base64 上传 | `/api/file-base64-upload` | 小文件、API 集成 | ≤10MB |
| 文件流上传 | `/api/file-stream-upload` | 大文件、本地文件 | 支持大文件 |
| URL 上传 | `/api/file-url-upload` | 远程资源迁移 | ≤100MB，30 秒超时 |

**返回**：`fileUrl`、`downloadUrl`，**文件保留 3 天**。  
**用途**：用户上传音频后，用 `fileUrl` 作为 `uploadUrl` 调用 upload-cover、add-vocals、add-instrumental、mashup 等。

## 三、参考创作方案（用户发音频按风格创作）

### 3.1 场景定义

用户提供一首参考歌曲的音频，希望按该风格创作一首**新歌**（新旋律、新歌词）。

### 3.2 方案对比

| 方案 | 实现方式 | 适用 | 前置条件 |
|------|----------|------|----------|
| **A. 上传并翻唱** | 用户上传参考曲 → 调用 upload-cover，prompt 填新歌词 | 保留原旋律、换风格 | 需 File Upload 获取 uploadUrl |
| **B. 风格分析 + 生成** | 用户上传 → 分析/手动选风格 → 填充创作参数 → 正常生成 | 全新创作，仅借鉴风格 | 可选：AI 风格分析 |
| **C. 混合** | 提供两种模式：翻唱模式（A）、参考创作模式（B） | 覆盖两种需求 | 同上 |

### 3.3 推荐：方案 C（双模式）

1. **翻唱模式**：上传参考曲 → 填写新歌词/风格 → 调用 upload-cover → 保留旋律、换风格。  
2. **参考创作模式**：上传参考曲 → 用户选风格标签或填写描述 → 填充到创作参数 → 走现有生成流程。

**入口**：在工作台增加「参考创作」Tab 或模式切换，与「从零创作」并列。

## 四、UI 优化建议

### 4.1 当前布局（已调整后）

```
[创作参数] | [生成任务] | [版本列表]
   左         中           右
```

- 任务与版本已强关联，点击任务右侧即显示该任务歌曲。
- 编辑任务已收起到版本列表底部「最近编辑」折叠区。

### 4.2 建议优化项

| 优化项 | 说明 | 优先级 |
|--------|------|--------|
| **创作模式切换** | 顶部增加「从零创作」\|「参考创作」切换，参考创作时显示上传区 + 风格选择 | 高 |
| **深度编辑分组** | 将 5 个能力按用途分组：变换(延长/替换)、分轨(人声分离)、导出(音乐视频)、人声( Persona ) | 中 |
| **替换分区时间选择** | 提供波形/时间轴预览或「按段落」选择（Verse1/Chorus），降低填秒数难度 | 中 |
| **风格提升入口** | 在创作参数「风格」旁增加「AI 优化风格」按钮，调用 boost-music-style | 低 |
| **积分/成本展示** | 若 API 支持，展示剩余积分、单次消耗，便于用户决策 | 低 |

## 五、功能优化建议

### 5.1 阶段二：uploadUrl 能力（需 File Upload）

**前置**：接入 Suno File Upload API，用户上传音频后获得 `fileUrl`。

| 功能 | 输入来源 | 说明 |
|------|----------|------|
| 上传并翻唱 | 用户上传 / 版本 audioUrl | 民族翻唱、参考曲换风格 |
| 上传并扩展 | 用户上传 / 版本 audioUrl | 拉长外部曲目 |
| 添加人声 | 人声分离的伴奏 / 用户上传伴奏 | 先分离或上传纯伴奏 |
| 添加伴奏 | 人声分离的人声 / 用户上传人声 | 先分离或上传干声 |

**注意**：平台内版本的 `audioUrl` 可能为本地或需鉴权，不一定可直接作 uploadUrl。需通过 File Upload 将音频上传到 Suno 侧获取可用 URL。

### 5.2 阶段三：进阶能力

| 功能 | 依赖 | 说明 |
|------|------|------|
| 生成混音 | 2 个 uploadUrl | 可从版本列表选 2 个版本，取其 audioUrl 经 File Upload 转成 uploadUrl |
| 提升音乐风格 | 无 | 创作参数区「风格」旁加按钮，调用 `/api/v1/style/generate` |
| WAV 转换 | taskId + audioId | 深度编辑中增加「导出 WAV」 |
| 从音频生成 MIDI | 人声分离分轨 | 分离后可增加「生成 MIDI」入口 |

### 5.3 一键预设（操作性增强）

| 预设 | 对应能力 | 参数示例 |
|------|----------|----------|
| 藏族风格翻唱 | 上传并翻唱 | style: Tibetan folk, Himalayan vocal... |
| 蒙古族风格翻唱 | 上传并翻唱 | style: Mongolian throat singing... |
| 卡拉OK 伴奏 | 人声分离 | type: separate_vocal → 输出伴奏 |
| 扩展 30 秒 | 延长音乐 | 从末尾续写 |

## 六、技术实现要点

### 6.1 File Upload 集成

- **Base URL**：文档示例为 `https://sunoapiorg.redpandaai.co`，需确认与 `api.sunoapi.org` 的关系及鉴权方式。
- **流程**：用户选择文件 → 调用 `/api/file-stream-upload` 或 Base64 → 获得 `fileUrl` → 作为 `uploadUrl` 传给 upload-cover 等。
- **过期**：文件 3 天内有效，需在 UI 提示用户及时使用。

### 6.2 编辑能力扩展

- 在 `editCapabilities.ts` 中增加 `uploadCover`、`uploadExtend`、`addVocals`、`addInstrumental`、`mashup`。
- 这些能力的 `inputType` 为 `upload_url`，需在调用前通过 File Upload 获取 URL。
- 平台内版本：可增加「转为 uploadUrl」步骤（上传到 Suno 获取临时 URL）。

### 6.3 数据模型

- `SongVersion` 可增加 `uploadSourceUrl`（用户上传来源），用于溯源。
- `EditTask` 已支持多能力，可扩展 `uploadUrl`、`uploadUrlList` 等字段。

## 七、分阶段实施路径

### 阶段一（已完成）

- 延长、替换、人声分离、Persona、音乐视频
- 本版本编辑结果展示（播放/下载/复制）
- 任务-版本强关联布局

### 阶段二（建议优先）

1. **File Upload 集成**：实现音频上传 API 路由，封装获取 `fileUrl` 的逻辑。
2. **参考创作入口**：工作台增加「参考创作」模式，支持上传 + 翻唱/参考风格。
3. **上传并翻唱**：接入 upload-cover，支持民族翻唱等预设。
4. **上传并扩展**：接入 upload-extend。
5. **添加人声/伴奏**：接入 add-vocals、add-instrumental（可与人声分离结果联动）。

### 阶段三（进阶）

1. 生成混音
2. 提升音乐风格（创作参数区）
3. WAV 转换
4. 替换分区时间选择体验优化
5. 积分/成本展示（若 API 支持）

---

## 八、待确认事项

1. **File Upload API**：Base URL 与鉴权是否与主 API 一致？需在开发前验证。
2. **参考创作**：是否同时支持「翻唱」与「参考风格创作」两种模式？
3. **公网部署**：若使用自建存储的 audioUrl，需确保 Suno 可访问；否则统一走 File Upload。
4. **阶段范围**：是否先完成阶段二中的 File Upload + 参考创作 + 上传并翻唱，再推进其他能力？

---

## 九、方案总结

| 维度 | 建议 |
|------|------|
| **API 覆盖** | 优先补齐 uploadUrl 类能力（上传翻唱、扩展、添加人声/伴奏），依赖 File Upload |
| **参考创作** | 新增「参考创作」模式，支持翻唱 + 参考风格两种路径 |
| **UI** | 创作模式切换、深度编辑分组、替换分区时间选择优化 |
| **操作性** | 民族翻唱等一键预设、风格提升入口、流程提示 |
| **不新增独立编辑页** | 参考创作与深度编辑均在工作台内完成，保持入口统一 |
