# 翻唱失败分析

## 〇、常见错误速查

| 错误码 | 错误信息 | 根因 | 解决方案 |
|--------|----------|------|----------|
| **413** | Uploaded audio matches existing work of art | 版权检测：参考曲与已有作品相似 | 使用原创或自录音频 |
| **500** | Internal Error, Please try again later | 服务端临时异常 | 稍后重试 |
| **超时** | 翻唱超时，请稍后在任务列表中查看 | 前端轮询超时，API 可能仍在处理 | 点击确定后刷新任务列表 |

## 一、413 版权检测错误（根因与解决）

### 错误信息

- **错误码**：413
- **错误信息**：Uploaded audio matches existing work of art.
- **来源**：Suno 内容识别系统（Audible Magic）

### 根因

Suno 与 Audible Magic 合作，在上传流程中接入内容识别技术。当检测到上传音频与已有版权作品相似时，会拒绝请求并返回 413。

**无法通过代码绕过**，这是版权合规机制。

### 官方依据（全网调研结论）

| 来源 | 说明 |
|------|------|
| [Audible Magic 官方新闻稿](https://www.audiblemagic.com/2024/10/18/ensuring-content-integrity-suno-partners-with-audible-magic-for-user-uploads/) | Suno 与 Audible Magic 正式合作，在用户上传流程中集成内容识别，**直接阻止版权录音**。Suno CEO 表示这是为了维护可信创意社区。 |
| [Suno 官方帮助 - Covers](https://help.suno.com/en/articles/2872257) | **「您只能对自己创作的歌曲使用此功能，不能用于他人作品。」** 官方明确限制 Cover 仅限用户自有内容。 |
| [Suno API 文档](https://docs.sunoapi.org/cn/suno-api/upload-and-cover-audio) | 文档中 413 仅写「主题或 prompt 过长」，**未单独说明音频版权检测**。说明 413 存在多种含义，文档不完整。 |
| [故障排查 - 音频处理问题](https://docs.sunoapi.org/cn/suno-api/upload-and-cover-audio-callbacks) | 提到「注意音频内容是否符合内容政策」，侧面印证内容合规检测存在。 |

**结论：不是我们配置错误。** 版权检测是 Suno 服务端强制策略，无 API 参数可关闭或绕过。

### 解决方案

1. **使用原创音频**：自己创作、录制的旋律或人声
2. **使用自录音频**：手机录音、即兴哼唱、乐器演奏等
3. **避免版权音乐**：不要使用已发行的商业歌曲作为参考曲

### 平台已做

- 检测到该错误时，展示中文提示：「检测到参考曲与已有版权作品相似，无法使用。请使用原创或自录音频。」
- 参考创作区域增加版权提示，提醒用户勿使用版权音乐

### 产品方向：如何满足「用户想用参考曲改编」的需求

用户需求「提供参考歌曲进行改编」很常见，但 Suno 不允许上传版权音乐。可行方向：

| 方向 | 说明 |
|------|------|
| **引导自录参考** | 提供「用手机录一段哼唱/旋律」作为参考的引导。用户哼唱、弹奏、即兴演唱，均为原创，可规避检测。 |
| **AI 生成参考曲** | 用户先用「定制创作」生成一首原创曲，再将该曲作为参考曲做「翻唱」。符合 Suno「只能用于自己创作的歌曲」的规则。 |
| **文案与预期管理** | 在参考创作入口明确说明：「参考曲仅支持原创或自录音频，不支持商业歌曲。」降低用户预期落差。 |
| **合法翻唱授权** | 若需合法使用他人歌曲，需获得机械授权、同步许可等。这是商业授权流程，需单独对接版权方。 |

## 三、翻唱流程与可能失败点

```
用户操作                    我们的平台                    Suno API
   │                           │                            │
   │  1. 选择/上传参考曲        │                            │
   │  ──────────────────────>  │  upload-audio (File Upload)  │
   │                           │  ─────────────────────────>│  上传到临时存储
   │                           │  <─────────────────────────│  返回 fileUrl
   │                           │                            │
   │  2. 点击「开始翻唱」       │                            │
   │  ──────────────────────>  │  upload-cover              │
   │                           │  ─────────────────────────>│  【可能失败点 A】
   │                           │  <─────────────────────────│  返回 taskId
   │                           │                            │
   │  3. 轮询任务状态           │  每 10 秒 poll              │
   │                           │  record-info               │
   │                           │  ─────────────────────────>│  【可能失败点 B】
   │                           │  <─────────────────────────│  返回 status
   │                           │  (重复直到 SUCCESS 或超时)   │
   │                           │                            │
   │  4. 收到 SUCCESS           │  创建版本、持久化媒体       │
   │                           │                            │
```

### 失败点 A：upload-cover（提交任务）

| 可能原因 | 说明 |
|----------|------|
| **413 版权检测** | 参考曲与已有版权作品相似，被 Audible Magic 识别并拒绝 |
| **uploadUrl 无效** | 参考曲上传失败或 URL 过期（3 天） |
| **音频格式/质量** | Suno 无法解析（损坏、格式不支持、时长超限） |
| **内容违规** | 歌词/风格触发内容审核 |
| **服务端过载** | Suno API 临时 500 |

### 失败点 B：record-info（轮询状态）

| 可能原因 | 说明 |
|----------|------|
| **任务处理中遇错** | Suno 在生成过程中内部异常，返回 500 |
| **服务端过载** | 查询时 API 临时不可用 |
| **taskId 异常** | 任务创建后服务端状态异常 |

## 四、500 错误的本质

**500 = HTTP 服务端内部错误**，表示：

- 不是我们传参错误（那是 400）
- 不是认证问题（那是 401/403）
- 是 **Suno API 服务端自身异常**：过载、上游故障、处理该任务时崩溃等

---

## 五、我们平台的改进点

### 1. 错误信息透传

当前 `sunoGetRecordInfo` 仅使用 `json.msg`，部分 API 返回 `message` 字段。应同时读取 `json.message`、`json.error`，确保用户看到真实错误信息。

### 2. 区分超时与 API 失败

- **超时**：10 分钟内未收到 SUCCESS，但 API 未返回错误 → 任务可能仍在处理
- **API 失败**：record-info 返回 500 或 status=FAILED → 任务已失败

建议在任务上区分这两种情况，UI 分别展示「超时待确认」与「失败」。

### 3. 失败任务保留 sunoTaskId

即使失败，也保存 `sunoTaskId`，便于用户稍后手动重试查询或提供给 API 方排查。

## 六、用户可采取的措施

### 413 版权检测

1. **更换参考曲**：使用原创或自录音频，避免版权音乐
2. **自录素材**：手机录音、哼唱、乐器演奏等

### 500 服务端错误

1. **稍后重试**：500 多为临时错误，间隔 10–30 分钟再试
2. **检查参考曲**：确保为 MP3/WAV，V4_5ALL 下时长 ≤1 分钟
3. **简化参数**：尝试更简短的歌词、风格描述
4. **联系 API 提供方**：若持续 500，提供 taskId 与时间点咨询
