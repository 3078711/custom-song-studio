# 编辑结果展示：流程分析与设计

## 一、创作音乐典型流程

```
填写创作参数 → 生成歌曲 → 试听选版 → 深度编辑 → 交付客户
     ↓              ↓           ↓           ↓          ↓
  左栏表单      中栏任务     右栏版本    延长/视频等   音频/视频/伴奏
```

## 二、当前断裂点

| 环节 | 现状 | 问题 |
|------|------|------|
| 生成任务 | 有列表，可点选查看版本 | ✅ 顺畅 |
| 版本列表 | 可试听、标记、发客户 | ✅ 顺畅 |
| 深度编辑 | 有按钮，可创建视频/人声分离 | ✅ 顺畅 |
| **编辑结果** | **仅 alert，数据存 localStorage** | ❌ **无处可寻，无法播放/下载/交付** |

**核心问题**：用户点了「创建音乐视频」→ 生成完成 → 找不到视频在哪，无法播放、下载、发给客户。流程在交付环节断裂。

## 三、设计原则

1. **紧贴操作上下文**：在哪个版本做的编辑，结果就展示在该版本下方，不另辟入口
2. **交付物优先**：播放、下载要明显，一键完成
3. **不增加操作步骤**：选版本 → 编辑 → 结果同屏可见，路径最短

## 四、推荐方案

### 4.1 主入口：本版本编辑结果（版本详情内）

**位置**：选中版本后，「深度编辑」按钮区下方

**内容**：展示该版本（`sourceVersionId`）产生的**已完成**编辑任务：

| 能力 | 展示方式 | 操作 |
|------|----------|------|
| 音乐视频 | 小预览 + 标题 | **播放**（内嵌/新窗口）、**下载** |
| 人声分离 | 人声 / 伴奏 两行 | 每项 **播放**、**下载** |
| 延长/替换 | 简短提示 | 「已生成新版本 V1-X」→ 跳转版本列表 |
| Persona | 简短提示 | 显示 personaId，可复制 |

**用户路径**：选版本 → 点「创建音乐视频」→ 确认 → 完成后**同一区域**即出现播放/下载，无需再找。

### 4.2 辅助入口：编辑任务记录（中栏）

**位置**：「生成任务」下方，可折叠

**内容**：最近 N 条编辑任务，按时间倒序。每项显示：能力、源版本、状态、完成时间；完成的可展开查看播放/下载。

**用途**：快速找到「我最近做的几个音乐视频」，无需在版本间翻找。

## 五、交付便捷性

- **播放**：音乐视频内嵌 `<video>` 或新标签打开；人声/伴奏用 `<audio>` 或新标签
- **下载**：`<a download href={url}>` 或 `window.open(url)`，文件名带版本号
- **复制链接**：可选，方便发给客户

## 六、结论

- 有助于创作流程：✅ 编辑结果可见、可交付，流程闭环
- 方便操作：✅ 结果紧贴版本，路径短
- 方便交付：✅ 播放/下载一键完成
- 专业度：✅ 交付物集中展示，符合音乐制作工作流
