！# Song Studio 平台优化方案 · 再评估

> **评估目标**：确保方案达到最优、创作者操作极便捷、强可扩展性、布局符合使用逻辑、所有功能均可实现（条件逐一明确）。

---

## 一、原方案不足与改进方向

### 1.1 原方案存在的问题

| 问题 | 说明 | 改进方向 |
|------|------|----------|
| **能力与输入割裂** | 上传类能力（添加人声/伴奏/混音）需 uploadUrl，但 UI 仅支持「选版本」 | 统一「输入来源」：版本 / 上传，能力根据输入自动启用/禁用 |
| **扩展性不足** | `EditCapabilityDef` 仅支持 `needsSunoIds`，无法表达 uploadUrl、多文件 | 扩展能力定义：`inputType`、`category`、`requiredInputs` |
| **未来扩展未预留** | 专业视频 API、专业编辑软件无占位 | 增加能力分类：`video`、`professional`，预留扩展区 |
| **条件不透明** | 用户不知「添加人声」需伴奏、「混音」需 2 个文件 | 每个能力展示「所需条件」，禁用时 tooltip 说明 |
| **布局与逻辑脱节** | 创作、改编、工具混在同一区域，无清晰分区 | 按「使用逻辑」分区：创作 → 改编 → 导出 → 扩展区 |

### 1.2 最优方案应满足的约束

1. **操作极便捷**：高频操作 ≤2 步到达，低频操作可发现
2. **强可扩展性**：新增能力 = 注册配置，新增分类 = 新增区块
3. **布局符合逻辑**：按用户心智「我想做什么」组织，而非按 API 组织
4. **全功能可达**：每个能力有明确 UI 路径，条件不满足时提示「需先…」

## 二、能力注册表扩展设计（可扩展性核心）

### 2.1 扩展后的能力定义

```ts
// 输入类型：决定能力需要什么
type InputType =
  | "suno_ids"      // taskId + audioId，仅 Suno 生成音频
  | "upload_url"    // 1 个 uploadUrl（上传或版本转 upload）
  | "upload_url_list"; // 2+ 个 uploadUrl（如混音）

// 能力分类：用于 UI 分组与未来扩展
type CapabilityCategory =
  | "create"        // 创作（从零、参考）
  | "edit_version"  // 改编-基于版本
  | "edit_upload"   // 改编-基于上传
  | "export"        // 导出（WAV、MIDI）
  | "tool"          // 辅助工具（风格提升、歌词生成）
  | "video"         // 视频制作（当前 Suno MV，未来专业视频 API）
  | "professional"; // 专业制作（未来 DAW、专业编辑软件）

interface EditCapabilityDef {
  id: string;
  label: string;
  description: string;
  confirmDetail: string;
  inputType: InputType;
  category: CapabilityCategory;
  /** 所需条件，用于 UI 展示与禁用提示 */
  requiredInputs: {
    /** 如 "平台版本（Suno 生成）" | "上传伴奏" | "2 个音频" */
    label: string;
    /** 条件不满足时的提示 */
    unmetHint: string;
  }[];
  needsSunoIds: boolean;  // 保留兼容
  async: boolean;
  /** 未来：关联外部服务 ID */
  externalServiceId?: string;
}
```

### 2.2 能力清单（含条件与分类）

| 能力 | category | inputType | 所需条件 | unmetHint |
|------|----------|-----------|----------|-----------|
| 从零创作 | create | — | prompt, style, title | — |
| 参考创作-翻唱 | create | upload_url | 上传参考曲 + 新歌词/风格/标题 | 请先上传参考曲 |
| 参考创作-参考风格 | create | upload_url | 上传参考曲 + 风格选择 | 请先上传参考曲 |
| 延长音乐 | edit_version | suno_ids | 平台版本 | 请选择 Suno 生成的版本 |
| 替换分区 | edit_version | suno_ids | 平台版本 + 时间区间 | 同上 |
| 人声分离 | edit_version | suno_ids | 平台版本 | 同上 |
| 生成 Persona | edit_version | suno_ids | 平台版本 | 同上 |
| 创建音乐视频 | edit_version | suno_ids | 平台版本 | 同上 |
| 上传并扩展 | edit_upload | upload_url | 上传音频 + 续写点/风格 | 请先上传音频 |
| 添加人声 | edit_upload | upload_url | 上传伴奏 + 歌词/风格 | 请先上传伴奏（或从人声分离获取） |
| 添加伴奏 | edit_upload | upload_url | 上传人声 + 风格标签 | 请先上传人声（或从人声分离获取） |
| 生成混音 | edit_upload | upload_url_list | 2 个音频 | 请选择或上传 2 个音频 |
| WAV 转换 | export | suno_ids | 平台版本 | 请选择 Suno 生成的版本 |
| 生成 MIDI | export | — | 人声分离分轨 URL | 需先完成人声分离 |
| 提升风格 | tool | — | 风格描述文本 | — |
| 歌词生成 | tool | — | 主题、风格 | — |
| 积分查询 | tool | — | — | — |
| **视频制作（未来）** | video | suno_ids / upload_url | 音轨 | 预留 |
| **专业编辑（未来）** | professional | — | 版本/导出 | 预留，可跳转外部或未来 API |

## 三、页面布局 · 使用逻辑优先

### 3.1 用户心智模型

```
我想… → 需要什么 → 怎么操作
├── 创作新歌 → 从零 / 参考曲 → 填参数 / 上传 + 填参数
├── 改编已有歌 → 版本 or 上传 → 选能力 + 填参数
├── 导出/分轨 → 版本 or 分离结果 → 选能力
└── 用工具 → 无 / 文本 → 直接操作
```

### 3.2 推荐布局：分区折叠 + 上下文驱动


### 3.3 操作便捷性设计

| 场景 | 操作步数 | 设计要点 |
|------|----------|----------|
| 从零创作 | 1 | 默认模式，参数区直接可见 |
| 参考翻唱 | 2 | 切参考 → 上传 + 填 → 开始 |
| 延长/替换/分离 | 2 | 选版本 → 点能力 → 确认 |
| 添加人声 | 2–3 | 上传伴奏 or 选分离结果 → 填歌词 → 执行 |
| 混音 | 3 | 选/上传 2 个 → 执行 |
| 风格提升 | 1 | 创作参数区旁一键 |

### 3.4 与现有布局的兼容

- **左栏**：由「创作参数」扩展为「创作 + 改编 + 工具 + 扩展」，采用**可折叠区块**
- **中栏**：生成任务，不变
- **右栏**：版本列表，不变；**选中版本**作为「改编」区的输入上下文
- **不新增页面**：所有能力在工作台内完成

## 四、输入来源统一与「版本→uploadUrl」转换

### 4.1 输入来源类型

| 类型 | 来源 | 适用能力 |
|------|------|----------|
| suno_ids | 平台版本（含 taskId、audioId） | 延长、替换、分离、Persona、MV、WAV |
| upload_url | 用户上传 / 版本经 File Upload 转 | 翻唱、扩展、添加人声、添加伴奏 |
| upload_url_list | 2 个 upload_url | 混音 |

### 4.2 版本 → uploadUrl 转换

- 平台版本的 `audioUrl` 可能为本地或需鉴权，Suno 无法直接拉取
- **方案**：用户选择版本后，点击「基于上传」类能力时，自动调用 File Upload 将 `audioUrl` 转为 `uploadUrl`
- 流程：选版本 → 点「添加人声」→ 后台 fetch audioUrl → 上传到 Suno → 获得 uploadUrl → 调用 add-vocals

### 4.3 人声分离结果 → 添加人声/伴奏

- 人声分离产出：`vocalUrl`、`instrumentalUrl`
- **添加人声**：可直接用 `instrumentalUrl`（若公网可访问）或经 File Upload 转
- **添加伴奏**：可直接用 `vocalUrl` 或经 File Upload 转
- UI：在「本版本编辑结果」的人声/伴奏旁增加「添加伴奏」「添加人声」快捷按钮

## 五、全功能可达性检查

### 5.1 能力 × 条件 × UI 路径

| 能力 | 所需条件 | UI 路径 | 条件不满足时 |
|------|----------|---------|--------------|
| 从零创作 | prompt/style/title | 创作区-从零 | 校验提示 |
| 参考翻唱 | 上传 + 歌词/风格/标题 | 创作区-参考-翻唱 | 置灰 +「请先上传」 |
| 参考风格 | 上传 + 风格 | 创作区-参考-风格 | 同上 |
| 延长 | 版本(sunoIds) | 改编区-选版本-延长 | 置灰 +「请选版本」 |
| 替换 | 版本 + 时间区间 | 改编区-选版本-替换 | 弹窗填秒数 |
| 人声分离 | 版本 | 改编区-选版本-分离 | 同上 |
| Persona | 版本 | 改编区-选版本-Persona | 同上 |
| 音乐视频 | 版本 | 改编区-选版本-MV | 同上 |
| WAV | 版本 | 改编区-选版本-WAV | 同上 |
| 上传扩展 | uploadUrl + 续写点 | 改编区-上传-扩展 | 置灰 +「请上传」 |
| 添加人声 | uploadUrl(伴奏) + 歌词 | 改编区-上传/分离结果 | 置灰 +「需伴奏」 |
| 添加伴奏 | uploadUrl(人声) + 风格 | 改编区-上传/分离结果 | 置灰 +「需人声」 |
| 混音 | 2×uploadUrl | 改编区-选 2 个/上传 2 个 | 置灰 +「需 2 个音频」 |
| 生成 MIDI | 分离分轨 URL | 分离结果区-生成 MIDI | 置灰 +「需先分离」 |
| 提升风格 | 风格文本 | 创作参数-风格旁按钮 | — |
| 歌词生成 | 主题/风格 | 辅助工具区 | — |
| 积分查询 | — | 辅助工具区 | — |

### 5.2 条件逐一添加的实现方式

- 每个能力在注册表中定义 `requiredInputs`
- 渲染时：检查当前上下文（selectedVersion、uploadedUrls、stems）是否满足
- 不满足：按钮 disabled + `title={unmetHint}`
- 满足：可点击，进入确认/表单流程

## 六、可扩展性：未来视频与专业编辑

### 6.1 视频制作扩展

- **当前**：Suno 音乐视频 → `category: "video"`，与 `edit_version` 并存
- **未来**：专业视频 API 接入 → 同属 `video`，在「扩展区-视频制作」增加新能力
- **实现**：在 `EditCapabilityDef` 增加 `externalServiceId`，UI 按 category 渲染，新能力仅需注册

### 6.2 专业编辑软件扩展

- **场景**：导出到 DAW、调用专业编辑 API
- **占位**：`category: "professional"`，扩展区「专业制作」
- **实现**：能力可配置 `actionType: "api" | "external_link"`，外部链接可跳转本地/云端专业软件

### 6.3 扩展架构示意

```
capabilityRegistry
├── create (创作)
├── edit_version (改编-版本)
├── edit_upload (改编-上传)
├── export (导出)
├── tool (辅助)
├── video (视频)      ← 未来可追加专业视频能力
└── professional (专业) ← 未来可追加 DAW、专业编辑
```

## 七、方案总结与实施建议

### 7.1 是否达到最优

| 维度 | 评估 | 说明 |
|------|------|------|
| 操作便捷 | ✅ | 高频 ≤2 步，输入来源统一，条件透明 |
| 可扩展性 | ✅ | 能力注册表 + 分类 + 扩展区，支持视频/专业编辑 |
| 布局逻辑 | ✅ | 创作 → 改编 → 工具 → 扩展，符合心智 |
| 全功能可达 | ✅ | 每能力有路径，条件不满足有提示 |

### 7.2 实施顺序建议

1. **Phase 1（架构）**：扩展 `editCapabilities.ts`，增加 `inputType`、`category`、`requiredInputs`
2. **Phase 2（布局）**：左栏改为分区折叠（创作 | 改编 | 工具 | 扩展）
3. **Phase 3（输入统一）**：实现「输入来源：版本 / 上传」，版本→uploadUrl 转换
4. **Phase 4（能力补齐）**：上传扩展、添加人声、添加伴奏、混音、WAV、风格提升、歌词生成
5. **Phase 5（扩展区）**：预留视频制作、专业制作区块，占位能力可先为「即将推出」

### 7.3 与原方案对比

| 项目 | 原方案 | 再评估方案 |
|------|--------|------------|
| 能力组织 | 按 API | 按输入类型 + 分类 |
| 上传类能力 | 分散 | 统一「编辑-上传」区块 |
| 条件提示 | 无 | 每能力 `requiredInputs` + unmetHint |
| 未来扩展 | 未预留 | video、professional 分类 + 扩展区 |
| 布局 | 创作+版本编辑 | 创作+改编(版本/上传)+工具+扩展 |
